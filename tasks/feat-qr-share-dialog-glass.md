# 大会共有（QR）ダイアログ限定グラスモフィズム

## 目的
- 「大会を共有（QR）」ダイアログに限定して、半透明+ブラー+薄枠のガラス表現を適用する。
- ライト/ダーク両テーマで本文・操作ボタンの可読性とQR読取性を維持する。

## 非目的
- 共有ダイアログ以外（一覧カード/詳細カード/フォーム/他ダイアログ）の見た目変更。
- 共有画像生成ロジック、共有URL、投稿文、Import/Export仕様の変更。
- Single-tab制御、Web Locks、Service Worker、DB/互換性に関する変更。

## 変更点
- 共有ダイアログ（`tournament-detail-share-dialog`）の `Paper` にグラス表現を適用する。
- 共有ダイアログの `Backdrop` をテーマ別濃度に調整する（任意推奨項目を反映）。
- QRプレビューを含むプレビュー枠を不透明寄りsurfaceで保護し、読取性低下を防ぐ。
- 共有ダイアログの `Paper` 角丸指定を `var(--mui-shape-borderRadius)` に調整し、見切れを防ぐ。
- 拡大プレビュー用ダイアログを `fullScreen` 化し、不透明背景 + 右上固定✕ + 中央contain表示へ変更する。
- ダークテーマ時の共有ダイアログ `Paper` に青みティントと `saturate` を加え、灰色化を抑えてグラス感を維持する。

## 影響範囲（ユーザー / データ / 互換性）
- ユーザー影響:
  - 「大会を共有（QR）」ダイアログの見た目が変化する。
  - 他画面・他ダイアログの見た目は維持する。
- データ影響:
  - 永続化データ、Payload、DBスキーマ変更なし。
- 互換性:
  - 共有・保存・コピー導線と既存イベント/API利用は変更しない。

## 実装方針（対象ファイル単位）
- `packages/web-app/src/pages/TournamentDetailPage.tsx`
  - 共有ダイアログの `PaperProps.sx` にテーマ分岐つきガラススタイルを追加。
  - 共有ダイアログの `slotProps.backdrop.sx` にテーマ別背景色を追加。
  - QR可読性ガードとして、プレビュー領域コンテナ背景を `theme.palette.background.paper` ベースに調整。
  - 共有ダイアログ `Paper` の `borderRadius` を CSS変数指定へ調整。
  - 拡大プレビュー `Dialog` を `fullScreen` + `theme.palette.background.default` 背景に変更。
  - 拡大プレビューに右上固定の円形✕ボタン（44px）を追加し、画像を flex center + contain で表示。
  - ダークテーマ時のみ `Paper` の `backgroundImage`（淡いティントグラデーション）と `backdropFilter: blur(12px) saturate(...)` を適用。

## テスト観点
- 共有ダイアログ表示時にのみ半透明/ブラー/薄枠が適用される（他ダイアログ非変更）。
- ライト/ダーク両テーマでタイトル・本文・ボタンの視認性が保たれる。
- QRプレビューの背景コントラスト確保により、読み取り可能性が維持される。
- 既存の共有機能（共有/画像保存/投稿文コピー/手動コピー表示）が回帰しない。
- 拡大プレビューはフルスクリーンで開き、右上✕で閉じられる。
- 拡大画像が画面中央で `object-fit: contain` 表示され、縦横比を維持して見切れない。
- ダークテーマでも共有ダイアログが灰色に沈まず、青みを帯びたグラス感を維持できる。

## ロールバック方針
- 単一コミットを `git revert` して共有ダイアログの見た目変更のみを戻せる状態にする。
- 回帰発生時は `TournamentDetailPage.tsx` のスタイル追加ブロックのみを優先的に撤回する。
